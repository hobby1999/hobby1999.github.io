Java反射 Java 反射的介绍 Java反射是Java语言中最重要的特性，可以说，Java语言的实现就是依靠了反射机制，从Class的加载，到Java虚拟机（Jvm、Dalvik、ART）的运行原理都强依赖于Java反射。 在经典教科书《Java核心技术 卷Ⅰ》中，对反射给出了如下定义： [能够分析类的能力的程序称为反射(reflective)] 也就是说Java程序中你可以对任何已知的、未知的类进行分析、调用，而这种能力是由通过在程序运行期间，Java运行时系统（Jvm Runtime）始终为所有对象维护一个运行时类标识符实现的，通过这个标识符，JVM可以跟踪每个对象所属的类，然后虚拟机利用这些信息正确的执行方法 注：您可以暂时将Java程序中的类想象成一个巨大的“数据库”，最初始的“Class”这个数据库中建立了许多张其他class的表，这些表中的id就是类标识符，反射就是通过这些id去获取到类相应的构造方法(Constructor)、成员变量（Field）、成员方法（Method）及类自身（Class），但当对Java虚拟机的深入了解，您必须摒弃这个想法，因为它并不严谨。 Java 反射在工程中的应用 反射在任何Java程序中都有非常重要的应用： 1、在Java工程中，可以动态的修改、访问其他类的一些变量来达到一些效果，比如检查某个类的成员变量是否合法。 2、在Web应用中，可以选择不同的数据库驱动器，如果当一个Web应用需要使用到多个数据库驱动器，那么可以在需要使用时在进行动态加载，反射功能能够很好的做到这一点。 3、在Android应用中，Android系统拥有庞大的API,但某些API被标识符标记上仅系统使用或不推荐应用使用，那么这个时候可以利用反射的方法调取这些API以供开发者使用。 Java反射的应用包括但不仅限于上面3点，许多稀奇古怪但便捷的操作都是通过反射完成，这使得Java工程在运行时获得了巨大的便利性 Java 反射的例子 下面我们将通过实际的代码展示Java反射的便利性，本文提供三种场景，与上文一一对应。
